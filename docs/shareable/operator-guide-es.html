Sección 01

Archivo Vintach — Guía para operadores

Esta guía cubre todo lo que un operador del marketplace necesita para gestionar Archivo Vintach: configurar contenido desde Sharetribe Console, configurar campos y categorías de publicación, importar publicaciones en masa, y entender la configuración de la aplicación. No se requiere experiencia técnica.

Sección 04

3. Campos de publicación

Los campos de publicación son los atributos que completan los vendedores al crear una publicación (color, size, brand, etc.).

Important — dónde vive cada campo: Archivo Vintach usa dos fuentes para campos de publicación. La mayoría se gestionan en Sharetribe Console, pero tres están definidos en el código de la aplicación and no pueden editarse en Console.

CampoDónde viveQuién puede editar
colorApp code (src/config/configListing.js)Solo equipo de desarrollo
all_sizesApp code (src/config/configListing.js)Solo equipo de desarrollo
brandApp code (src/config/configListing.js)Solo equipo de desarrollo
generoConsole → Build → Listing fieldsOperador
estadoConsole → Build → Listing fieldsOperador
estiloConsole → Build → Listing fieldsOperador

Cuando existe un campo con la misma clave en Console y en código, la definición en código tiene prioridad. So adding color, all_sizes, or brand in the Console no tiene efecto — la app ignorará la versión de Console y usará la versión en código. To add, rename, or remove options for these three fields, pide al equipo de desarrollo que actualice src/config/configListing.js.

Las secciones siguientes describen la configuración canónica. For the three code-only fields, the tables document what the app expects; no necesitas recrearlos en Console.

Los campos gestionados en Console (Console → Build → Listing fields) tienen:

  • Key — el ID interno (no cambia una vez creado)
  • Schema type — enum (elige uno) or multi_enum (elige varios)
  • Scope — always public for searchable fields
  • Options — los valores que pueden elegir los vendedores

Después de crear los campos, go to Console → Build → Listing types, abre tu tipo de publicación, and add the fields there para que aparezcan en el formulario de publicación.

Sección 05

4. Secciones de la landing page

La landing page se construye en Console → Content → Pages → Landing Page. Cada sección que agregas crea un bloque visual en la página.

Cómo funcionan las secciones

Cada sección tiene:

  • Section Type — determina qué hace la sección (the "template")
  • Section ID — un nombre único para esta sección específica
  • Section Name — nombre visible opcional; también se usa para aplicar tokens de estilo visual (see Section Display Options)
  • Blocks — elementos individuales dentro de la sección (listings, categories, images, etc.)
  • Block Name — lo usan las secciones personalizadas para pasar datos (no es una etiqueta visible)

4.1 Hand-Picked Listing Carousel (avSelections)

Muestra un carrusel horizontal desplazable of publicaciones específicas que eliges manualmente.

Setup:

  1. Agrega una sección. Define Section Type como avSelections.
  2. Define Section ID como av-selections or av-selections-[any-name] (e.g. av-selections-verano). Cada ID único es un carrusel independiente.
  3. Agrega un bloque por publicación. In each block, set Block Name to the listing's UUID. - Find a listing's UUID in Console → Manage → Listings → open the listing → copia el ID desde la URL.
  4. Opcionalmente agrega Title, Descripción y Call to Action to the section header.

Notas:

  • Solo aparecen publicaciones publicadas. Los borradores y publicaciones cerradas se omiten silenciosamente.
  • El orden de los bloques controla el orden de las publicaciones en el carrusel.
  • Puedes tener múltiples av-selections-* secciones en la misma página.

4.2 Category Card Carousel (avSelectedCats)

Muestra un carrusel horizontal de tarjetas de categoría. Each card shows a photo and category name, and al hacer clic abre la página de búsqueda filtrada por esa categoría. No hay carga automática de datos — todo el contenido viene de lo que ingresas en Console.

Setup:

  1. Define Section Type como avSelectedCats.
  2. Define Section ID como av-selected-cats or av-selected-cats-[name].
  3. Agrega un bloque por categoría: - Block Name — the category ID (e.g. ropa, ropa-vestidos, bolsas, zapatos) - Title — nombre visible opcional. Si se omite, la app usa la categoría's name from your category configuration. - Media — sube la foto que se muestra en la tarjeta.

Notas:

  • The category ID in Block Name debe coincidir exactamente the ID in your category configuration.
  • Las tarjetas enlazan a la página de búsqueda filtrada por esa categoría.

4.3 Tag or Category Filtered Carousel (avTagListings)

Displays a carousel of listings cargadas automáticamente por un filtro. The filter está definido en the first block's Block Name.

Setup:

  1. Define Section Type como avTagListings.
  2. Define Section ID como av-tag-listings-[name] (e.g. av-tag-listings-hot).
  3. Add one block. Set its Block Name usando uno de estos formatos:
Formato de Block NameQué carga
tag:hot-listPublicaciones etiquetadas hot-list
cat:ropa-vestidosPublicaciones en la categoría ropa-vestidos category
hot-listIgual que tag:hot-list (la etiqueta es el valor predeterminado)

Notas:

  • Muestra hasta 24 publicaciones publicadas.
  • Solo el primer bloque's Block Name is used as the filter. Los bloques adicionales se ignoran.
  • Puedes tener múltiples av-tag-listings-* sections, each con su propio filtro.

4.4 Recommended Listings Grid (avRecommendeds)

Displays una cuadrícula de varias columnas of publicaciones seleccionadas manualmente. Funciona igual que el carrusel pero se muestra como cuadrícula.

Setup:

  1. Define Section ID como exactly av-recommendeds (esto es fijo — solo una por página).
  2. Define Section Type como avRecommendeds.
  3. Add one block per listing, setting Block Name to the listing's UUID.

4.5 User Profile Carousel (avSelectedUsers)

Displays a horizontal carousel of tarjetas de perfil de usuario. Útil para destacar vendedores o socios de marca.

Setup:

  1. Define Section Type como avSelectedUsers.
  2. Define Section ID como av-selected-users or av-selected-users-[name].
  3. Agrega un bloque por usuario. Set Block Name to the user's UUID. - Find a user's UUID in Console → Manage → Users → open the user → copia el ID desde la URL.

4.6 Instagram-Style Image Grid (avInstaGrid)

Displays a responsive cuadrícula de fotos con imágenes que subes directamente en Console. Ideal para galerías lookbook o mood boards. No se cargan datos de publicaciones.

Setup:

  1. Define Section Type como avInstaGrid.
  2. Define Section ID como av-insta-grid or av-insta-grid-[name].
  3. Add one block per image. Sube la imagen in the block's Media field. El título y la descripción del bloque son overlays opcionales.

Comportamiento de columnas por tamaño de pantalla:

Ancho de pantallaNúmero de columnas
Muy pequeña (menor a 550 px)2
Pequeña (550–767 px)3
Mediana (768–1023 px)4
Grande (1024 px o más)6

4.7 Hero Banner (Standard) (avHero)

El hero banner principal con gradiente en la parte superior de la landing page. Solo uno por página.

Setup:

  1. Define Section Type como avHero.
  2. Define Section ID como av-hero.
  3. Completa Title, Descripción y Call to Action in the section fields.
  4. Configura una imagen de fondo via the section's ajustes de apariencia.

4.8 Hero Banner (Multi-Instance) (avHero2)

Un hero banner flexible that admite dos botones CTA and an una imagen móvil diferente opcional. Unlike the standard hero, pueden existir múltiples instancias independientes en la misma página.

Setup:

  1. Define Section Type como avHero2.
  2. Define Section ID como av-hero2-[unique-name] (e.g. av-hero2-summer).
  3. Fill in Title, Descripción, o más to two Call to Action buttons.
  4. Set the background image in the section's ajustes de apariencia.

Opcional — imagen diferente en móvil: Ve a Console → Content → Translations and add a key:

AVHero2.[your-section-id].mobileBackgroundUrl = (image URL)

Opcional — hacer que toda la sección sea un enlace clicable: Add:

AVHero2.[your-section-id].bgLink = (la ruta URL, e.g. /s?pub_categoryLevel1=ropa)

4.9 Block-Based Hero (avHero3)

Un hero donde cada bloque define una franja de imagen with an overlay de texto opcional. Ideal para layouts hero de múltiples paneles.

Setup:

  1. Define Section Type como avHero3.
  2. Define Section ID como av-hero3-[name].
  3. Add one or more blocks. Each block provides a Media (background image) and optional Title / Descripción as a text overlay.

4.10 Video + Text Split (avVideo)

Una sección de ancho completo dividida en dos mitades: un video con reproducción automática a la izquierda, and título, descripción y CTA a la derecha. En móvil, las mitades se apilan verticalmente.

Setup:

  1. Define Section Type como avVideo.
  2. Define Section ID como av-video-[name].
  3. Completa Title, Descripción y Call to Action for the right side.
  4. Para configurar la URL del video, go to Console → Content → Translations and add:

AVVideo.[your-section-id].videoUrl = (URL directa del archivo de video, e.g. an MP4 link)

Notas:

  • El video se reproduce automáticamente en silencio y en bucle.
  • Las políticas de autoplay del navegador pueden impedir la reproducción automática en algunos dispositivos si el video no está silenciado.

4.11 Pricing Plans (price-columns)

Muestra el selector interactivo de planes de precios with selector mensual/anual.

Setup:

  1. Define Section Type como price-columns.
  2. El contenido (plan names, prices, features, CTAs) se carga desde un archivo de datos alojado. Ve a Console → Content → Assets and crea un archivo en la ruta content/pricing-plans.json with your plan data. Pide al equipo de desarrollo el formato exacto.

5. Barra de navegación

The top navigation bar has three dropdown menus on desktop:

  • Ropa — clothing subcategories
  • Accesorios — accessories, bags, shoes
  • Marcas — automatically built from the brand field options

Editar desplegables 1 y 2 (Ropa y Accesorios)

These dropdowns are managed in a file called top-bar.json in the app. To change the categories shown, pide al equipo de desarrollo que actualice this file. The current configuration shows:

Ropa dropdown: Ver Todo, Tops, Camisetas, Camisas, Chamarras/Sacos, Pantalones, Jeans, Faldas, Vestidos, Ropa Deportiva, Jumpsuits, Sets

Accesorios dropdown: Ver Todo, Bolsas, Zapatos, Cinturones, Gorras/Gorros, Joyería

Desplegable 3 — Marcas (automático)

This dropdown builds itself automatically from the brand field options in the app configuration. You do not need to configure it — it always reflects the current brand list.

Sección 07

6. Hot List

The Hot List is a curated carousel of featured listings. Any published listing tagged with hot-list automatically appears in it.

Agregar una publicación a Hot List

Via Sharetribe Console:

  1. Ve a Console → Manage → Listings.
  2. Open the listing.
  3. In the Extended Data / Public Data section, find the tags field and add hot-list.
  4. Save.

Via the bulk import tool: Add hot-list to the pd_tags column for any listing row in your CSV.

Quitar una publicación de Hot List

Remove hot-list from the listing's tags field. The listing will no longer appear in the Hot List on the next page load.

Crear carruseles curados adicionales

You can create as many tag-based carousels as you want. For example, to create a "New Arrivals" carousel:

  1. Tag the relevant listings with nueva-llegada in their public data.
  2. Add a avTagListings section to the landing page with Section ID av-tag-listings-nuevas.
  3. Add one block with Block Name nueva-llegada.
Sección 08

7. Herramienta de importación masiva

7.1 Resumen

The bulk import tool lets you create many listings at once by uploading a single ZIP file that contains a CSV spreadsheet and all the listing images. It is available at /admin/bulk-import on the marketplace.

Key facts:

  • Up to 100 listings per import
  • Up to 400 images per ZIP file
  • Maximum ZIP file size: 50 MB
  • Each row in the CSV creates one listing
  • Rows are processed one by one — if one row fails, the others continue
  • You must be signed in with an operator account allowlisted by the development team

7.2 Cómo usarla

  1. Open the tool — go to [your marketplace URL]/admin/bulk-import.
  2. Confirm you are signed in as an allowlisted operator — there is no import key to enter or store in the browser.
  3. Download the CSV template — click "Download CSV Template" to get a blank spreadsheet with all the correct column headers.
  4. Fill in the CSV — one row per listing. See the CSV Column Reference below.
  5. Prepare your images — name each image file clearly. Image filenames in the CSV debe coincidir exactamente the filename (including extension) of the image files in your ZIP.
  6. Create the ZIP — pack your completed CSV and all image files into a single .zip archive. Images can be in subfolders.
  7. Upload and start — click "Select ZIP file", choose your archive, then click "Start Import".
  8. Monitor progress — the page shows a live progress bar and reports which listings were created successfully and which rows had errors.

7.3 Estructura del archivo ZIP

Your ZIP must contain exactly one CSV file (at any level inside the archive) and all the images referenced by that CSV.

Ejemplo:

code
my-listings.zip
├── listings.csv
├── vestido-frente.jpg
├── vestido-trasera.jpg
└── photos/
    ├── jeans-frente.jpg
    └── jeans-detalle.jpg

ZIP rules:

  • Exactly one CSV file (any filename, any folder level)
  • All images referenced in the CSV must be present
  • Two images cannot share the same filename, even in different folders
  • Maximum 401 entries (1 CSV + up to 400 images)
  • Maximum 50 MB compressed upload size

Sección 10

9. Textos personalizados de traducción

Custom translation strings control operator-editable text throughout the app. They live in two places:

  • App translation files (en_av.json / es_av.json) — the baseline defaults shipped with the code. Updated by the development team.
  • Sharetribe Console → Content → Translations — operator overrides. Console values take precedence over the app files.

Cómo actualizar un texto en Console

  1. Ve a Console → Content → Translations.
  2. Use the search box to find the key you want (e.g. AVWelcomePopup.vendedor.title).
  3. If the key already exists, click it and edit the value in the text field, then save.
  4. If the key does not appear yet (common for per-section dynamic keys like AVHero2.my-section.mobileBackgroundUrl), click Add translation, enter the key exactly as shown in the tables below, type the value, and save.

Changes take effect on the next page load. A key left blank or not added at all causes that element to be silently hidden — the app never shows a raw key string to users.

Two languages: Add the same key in both Inglés and Español tabs, or whichever languages your marketplace supports.
